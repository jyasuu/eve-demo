執行 GitHub Trending 每日摘要，為每個上榜專案產出繁體中文摘要，並「逐一」發送到 Discord 頻道（每個專案一則訊息，不要合併成一則長訊息）。

先確認今日日期（可用 `date +%Y年%m月%d日`）。

## 模組 A：擷取 Trending 清單

1. `web_fetch` 抓取 `https://github.com/trending`（預設不加語言篩選，抓「Today」區間）
2. 若內容為 HTML 而非乾淨文字，改用 `bash_tool` 搭配 Python（`beautifulsoup4` 或正則）解析，萃取每個專案的：
   - 專案全名（`owner/repo`）
   - 專案連結（`https://github.com/owner/repo`）
   - 一句話英文描述（repo 原本的 description）
   - 主要語言
   - 累積 star 數、今日新增 star 數（"X stars today"）
3. 僅取前 10 名（若使用者要求更多，依指示調整），依上榜順序排列成清單，供後續模組逐一處理

## 模組 B：逐專案中文摘要

對模組 A 清單中的**每一個專案**，重複以下步驟（一次處理一個，不要批次一起做）：

1. 若原本的 description 過於精簡或看不出用途，`web_fetch` 該專案的 GitHub 頁面（或 README）補充脈絡
2. 產出繁體中文摘要，包含：
   - **專案名稱與一句話定位**（這個專案是做什麼的）
   - **核心功能／解決的問題**（2～3 點）
   - **技術棧／主要語言**
   - **適合誰使用**（目標受眾或使用情境）
   - **今日熱度**：star 數與今日新增 star 數
3. 摘要控制在 100～150 字繁體中文，語氣清楚、簡潔，不要逐字翻譯英文 README，用自己的話重寫

## 模組 C：逐一發送 Discord

**重點：每個專案獨立呼叫一次 `send_discord_message`，不要把整份清單合併成一則訊息。**

1. 第一則訊息先發送當日標題，例如：
   `📈 GitHub Trending 每日精選 — [今日日期]（共 N 個專案）`
2. 接著針對模組 B 產出的每個專案摘要，依上榜排名順序，逐一呼叫 `send_discord_message`，格式建議：

   ```
   #[排名] [owner/repo]
   🔗 https://github.com/owner/repo
   ⭐ [累積 star] · 🔥 今日 +[今日新增 star] · 🏷️ [主要語言]

   [繁體中文摘要內容]
   ```

3. 每則訊息發送後，間隔約 1～2 秒再發送下一則，避免觸發 Discord rate limit
4. 若某專案摘要產生失敗或抓取不到資料，仍發送該專案的基本資訊（名稱、連結、star 數），並註明「摘要暫缺」，不要整段跳過

## 收尾

全部專案發送完畢後，發送最後一則總結訊息，例如今日語言分布或熱度最高的專案提醒。
